# ECP runtime Python dependencies.
#
# v1 had zero runtime third-party packages — the markdown-emission audit pipeline
# parsed FINDING blocks via regex and emitted prose directly to disk. v2's Layer 2
# structuring requires JSON Schema validation against schema/cluster-emission-v1.json
# and schema/finding-v1.json (both draft 2020-12 with conditional if/then/else
# rules), and the only Python validator with full draft 2020-12 support is
# `jsonschema`. fastjsonschema is draft 04/06/07 only; pydantic would force a
# schema rewrite.
#
# Tight pin avoids surprise major-version jumps; the 4.x line has been stable
# since 4.18 (referencing.Registry pattern) and the 4.26 minimum guarantees
# referencing is bundled and the format-checker is mature. See Phase B (Phase B
# brings forward this dep so test-specialist.py can validate emissions in
# isolation; Phase E extends usage to the structuring layer).
#
# Authored 2026-04-27. See:
# - docs/plans/2026-04-27-feat-ecp-v2-redesign-plan.md Phase E.1
# - docs/brainstorms/2026-04-27-ecp-v2-architecture-brainstorm.md (validator selection)
